# Claude Code 轨迹采集工作台作业流程 (v3.1 release)

工作台地址：`http://127.0.0.1:5177`

## 1. 启动工作台

```bash
cd <项目目录>
npm run workbench
```

浏览器打开 `http://127.0.0.1:5177`。

首次使用需先生成证书：`npm run setup`（只需一次）。

## 2. 创建 Session

首次进入页面会弹窗提示新建 Session。输入名字则用名字作为目录名，留空则自动用时间戳（`20260705_145500`）。

侧边栏可随时：
- **重命名** — 目录名同步修改
- **删除** — 删除整个 Session 目录
- **打开目录** — 资源管理器打开 Session 文件夹
- **从已有 Session 导入** — 复制其他 Session 的抓包+历史

## 3. 启动代理

在工作区 A 的代理配置面板：

- 代理端口：默认 `8888`
- TARGET_HOST：留空则拦截所有 HTTPS 请求

点击 **启动代理**，按钮变绿显示"代理中"。

## 4. 一键启动 Claude Code（推荐）

代理面板下方有三个按钮，点击即可**自动跳转工作区 C 并在对应终端中设置代理并启动 Claude Code**：

| 按钮 | 终端 | 说明 |
|---|---|---|
| ▶ 在 Git Bash 终端启动 | Git Bash (MSYS2) | `export` 语法设置环境变量 |
| ▶ 在 PowerShell 终端启动 | PowerShell | `$env:` 语法设置环境变量 |
| ▶ 在 CMD 终端启动 | 命令提示符 | `set` 语法设置环境变量 |

点击后会自动：跳转到工作区 C → 切换到对应 Shell → 设置代理环境变量 → 执行 `claude --permission-mode bypassPermissions`。

> 无需手动复制粘贴，全程在浏览器内完成。

## 5. 工作区 C：内置终端

工作区 C 提供完整的终端环境（基于 PTY 伪终端），无需离开浏览器即可操作命令行。

### Shell 切换

右上角三个按钮可快捷切换 Shell：

| 按钮 | Shell | 适用场景 |
|---|---|---|
| Git Bash | `bash --login` | Unix 风格命令，默认 |
| PowerShell | `powershell -NoLogo` | Windows 原生脚本 |
| CMD | `cmd.exe` | 传统 Windows 命令 |

### 快捷键

| 快捷键 | 功能 |
|---|---|
| Ctrl+C（无选区） | 发送 SIGINT 中断当前进程 |
| Ctrl+V | 粘贴剪贴板 |
| Ctrl+Shift+V | 粘贴 |
| Shift+Insert | 粘贴 |

### 特性

- **PTY 伪终端**：支持嵌套 Shell（如 `cmd` 里启动 `powershell`）、交互式程序
- **WebGL 渲染**：GPU 加速，大量输出不卡顿，中文 IME 输入正常
- **自适应尺寸**：终端宽度自动匹配页面，窗口缩放时实时同步
- **三色主题**：工作区 A（蓝）、B（粉）、C（青绿）独立配色

## 6. 进行 Claude Code 编程任务

正式采集要求：
- 真实编程任务（需求分析 → 读文件 → 改代码 → 调试 → 修复）
- 启用 Extended Thinking
- 包含多种工具调用（ReadFile、WriteFile、RunCommand 等）

## 7. 停止代理并查看数据

任务结束后：
1. 点击 **停止代理**
2. 点击 **刷新抓包**

抓包表格显示每轮请求的状态、模型、耗时、Tokens、工具调用等。点击行可查看完整内容。

## 8. 导入数据

### 从已有 Session 导入

侧边栏 Session 卡片中点击 **⇩ 从已有 Session 导入**，选择一个 Session 即可将其抓包和历史一键导入当前 Session。

### 从 Claude Code 历史导入

在工作区 A 的 **Session 数据管理** 面板：

- **从 Claude Code 历史导入**：扫描 `~/.claude/projects/`，选一个 JSONL 导入
- **清除工作台显示**：重置页面显示，不删除文件

## 9. 运行验证

点击 **运行验证**。

验证结果展示：
- 匹配率（圆环图）
- 回复一致 / 工具一致 / Thinking / 失败请求（标签）
- 逐轮对比明细表

全绿表示验证通过。

## 10. SOP 转换（工作区 B）

切到工作区 B，填写元数据：

| 字段 | 说明 |
|---|---|
| Instance ID | 留空自动生成 |
| Task ID | 留空自动生成 |
| Model | 留空自动映射，或手动选 opus-4-6/4-8 |
| Language | 留空自动检测 |
| Repo | 必填，代码仓库地址 |
| Base Commit | 必填，任务开始时的 commit hash |
| Problem Statement | 必填，问题描述 |
| Summary CoT | 留空自动生成草稿 |

点击 **生成 SOP JSONL + 质检**。

## 11. 质检报告

生成后自动显示 QC 报告：
- ✅ 通过 / ❌ 未通过
- Errors：必须修复的问题
- Warnings：建议关注
- Info：统计信息

## 12. 下载交付

两个下载按钮：
- **下载 trajectory.jsonl** — 交互轨迹
- **下载 instance.json** — 任务元数据

## 13. 主题设置

点击顶栏 🎨 按钮可自定义：
- 壁纸选择
- 卡片透明度
- 壁纸亮度和模糊

设置自动保存，刷新不丢。

## 快速版流程

```
新建 Session → 启动代理 → 一键启动 Claude Code（自动跳转终端）
→ 编程任务 → 停止代理 → 导入数据 → 验证 → 填元数据 → 生成 → 下载交付
```
